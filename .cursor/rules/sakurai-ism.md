# 設計思想
桜井政博氏の設計哲学を継承。詳細は `.cursor/skills/sakurai-ism/SKILL.md` を参照。

# 絶対原則（常に適用）
- 60fps死守
- 作り手の都合をプレイヤーに押し付けない
- 迷ったら「面白さの核」に近い案を選べ

# コード規約
- フィードバック演出（ヒットストップ、画面振動）は省略禁止
- UI文字を減らし、アイコン・色で伝える

3. プレイヤー体験を磨き上げる (Polishing the Player Experience)
ゲームの核となるロジックやシステムがいかに優れていても、それだけでは最高のプレイヤー体験は生まれません。ロジック上の面白さを、演出や操作感を通じてプレイヤーの感情に直接訴えかけ、心地よく伝えること。それこそが、凡作と名作を分ける最後の、そして最も重要な工程です。この章では、体験の質を極限まで高めるための具体的な手法に焦点を当てます。
3.1. 触り心地を決定づける効果音 (Sound Effects that Define the Feel)
桜井氏が掲げる「名作に音が悪いものなし」という持論は、サウンド、特に効果音（SE）の重要性を端的に示しています。プレイヤーの操作に対する直接的なフィードバックである効果音は、ゲームの「触り心地」そのものを定義します。心地よい効果音を設計するには、以下の2つの原則が極めて重要です。
• アタック強め 音の出だしを強く、鋭く、素早くすることで、プレイヤーの入力に対する即時的な反応を返します。これにより、操作の手応えと確かな感触が生まれます。
• 残響はほどほどに 音の余韻を引きずりすぎないように設計します。特にパズルゲームのように連続した操作が想定される場合、一つ一つの音が前の音の残響に埋もれてしまっては、小気味よいテンポ感が失われます。各操作が明瞭に聞こえることが重要です。
「ワンこねくと」において、パズルピースを動かし、配置し、消去する一連の操作にこれらの原則を適用することで、単なるロジックパズルを超えた、触っているだけで気持ちの良い、中毒性の高い体験を生み出すことができるでしょう。
3.2. 手応えを演出するエフェクト (Effects that Create a Sense of Impact)
エフェクトは、プレイヤーのアクションが世界に影響を与えたことを視覚的に伝えるための重要な演出です。しかし、単に光らせれば良いというものではありません。デザインの鍵は「メリハリ」にあります。
「色加算」に頼りすぎた表現は、全体がぼんやりとした印象になりがちで、手応えが薄れてしまいます。エフェクトはどうしても明るめになりがちなので、輪郭のハッキリした「パリッとしたエフェクト」や、あえて「暗いものを混ぜる」手法が効果的です。これにより明るい部分がより際立ち、シャープでインパクトのある印象を与えることができます。
また、大きなアクション、例えば「ワンこねくと」で大規模な連鎖消しが成功した際には、画面全体を揺らす「画面振動（スクリーンシェイク）」を積極的に導入すべきです。これはプレイヤーに絶大な手応えと爽快感を与える、非常に効果的な手法です。
3.3. プレイヤーを褒め称える演出 (Presentation that Praises the Player)
プレイヤーにご褒美を与えるだけでなく、その行動自体を「褒める」演出は、プレイヤーのモチベーションを飛躍的に高めます。『真・三國無双』シリーズで味方武将がプレイヤーの活躍を賞賛するセリフや、『大乱闘スマッシュブラザーズ SPECIAL』で相手を撃墜した際に舞う華やかな紙吹雪は、その好例です。これらはシステム上の報酬とは別に、プレイヤーの感情に寄り添い、高揚感を生み出します。
「ワンこねくと」においても、プレイヤーを「褒め殺す」くらいの気概で以下の演出を実装します。
• 高コンボ達成時: 派手なエフェクトと共に「Excellent!」「Marvelous!」といった賞賛のテキストを表示する。
• 難関ステージクリア時: キャラクターがプレイヤーを称える特別なアニメーションを再生する。
• 記録更新時: ファンファーレと共に画面が祝福ムードに包まれる演出を導入する。
こうした演出は、プレイヤーに「自分のプレイが認められている」という感覚を与え、次なる挑戦への意欲を掻き立てます。
3.4. 実践で学ばせるチュートリアル (Tutorials that Teach Through Practice)
優れたチュートリアルとは、プレイヤーを無機質な練習ステージに閉じ込めるものではありません。最も効果的なのは、ゲームの導入部で「いきなり実践に飛び込ませ、戦わせながら学ばせる」方法です。
プレイヤーは早くゲーム本編をプレイしたいと望んでいます。その気持ちを削ぐことなく、自然なゲームプレイの流れの中で、必要な操作やルールを習得させる設計が理想です。例えば、特定のスキルを使わなければ進めないような地形を序盤に配置することで、プレイヤーはテキストを読むのではなく、自らの手で操作を試し、その意味を体感的に理解することができます。チュートリアルが「作業」に感じられないよう、ゲーム本編とシームレスに融合させる工夫が求められます。
最高のプレイヤー体験を構築するためには、それを生み出すチームが円滑に機能することが不可欠です。最終章では、これらの理想を実現するための、効率的かつ健全なチーム運営の哲学について論じます。
